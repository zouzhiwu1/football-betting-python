# 系统架构说明

系统分为三个模块，本仓库（football-betting-pipeline）仅实现 **模块 1**。

---

## 模块 1：数据抓取与曲线图生成（本仓库）

- **职责**：抓取智云比分数据 → 合并、计算 → 生成曲线图图片。
- **产出**：按日期目录存放的曲线图文件，例如  
  `{CRAWLER_DOWNLOAD_DIR}/{YYYYMMDD}/{主队}_VS_{客队}_曲线.png`
- **运行方式**：定时任务执行 `main.py`（crawl → merge_data → calc_car → plot_car），或单独运行各脚本。
- **与模块 2 的关系**：模块 2 可读取上述目录中的图片，对外提供查询接口或页面。

---

## 模块 2：用户管理系统

- **职责**：用户注册、登录、支付、**查询**（按日期/球队名查曲线图等）。
- **查询功能**：即「按日期 + 球队名搜索并展示曲线图」，已部署在用户管理系统（如 football-betting-platform）内，与注册、登录、支付统一入口。
- **与模块 1 的关系**：读取模块 1 生成的曲线图目录（或由模块 1 写入的共享存储），对外提供 API 与查询页。
- **与模块 3 的关系**：为 React 手机客户端提供后端 API。

---

## 模块 3：手机客户端

- **实现**：React。
- **职责**：面向用户的移动端界面，所有数据与操作通过 **模块 2（用户管理系统）** 的接口完成，不直接访问模块 1。

---

## 数据流简图

```
[定时任务] → 模块 1 (本仓库) → 曲线图图片 (按日期/比赛)
                    ↓
              模块 2 读取图片，提供查询 API/页面
                    ↓
              模块 3 (React) 调用模块 2 接口
```
